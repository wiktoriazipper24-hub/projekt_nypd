Profiling analysis using SnakeViz revealed that the primary bottleneck in the program is the analyze_lines function (stats.py), which took 36.4 s to execute. A detailed look at the call stack shows that a significant portion of this time (16.3 s) is spent executing the update method from the built-in collections.Counter module. This performance issue arises from calling letter_counter.update() inside a loop for every single line or extracted word, which creates massive function call overhead when processing large datasets in Python.
